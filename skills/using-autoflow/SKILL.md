# AutoFlow Initialization Skill

Bootstrap a new or existing project with the AutoFlow test automation framework.
Guides through environment validation, tech stack selection, repo wiring, and
connection configuration, then generates the project scaffold and CLAUDE.md.

---

## Step 1: Environment Detection

Check that all required tools are available and report their status.

```bash
python3 --version   # require >= 3.12
uv --version        # require uv package manager
git --version       # require git
```

Also detect whether this is an existing project:

```bash
test -f pyproject.toml && echo "existing" || echo "new"
```

Output a status table:

| Tool      | Required | Found | Status |
|-----------|----------|-------|--------|
| Python    | >= 3.12  | x.y.z | OK/FAIL |
| uv        | any      | x.y.z | OK/FAIL |
| git       | any      | x.y.z | OK/FAIL |
| pyproject | —        | yes/no | INFO |

If any required tool is missing, print installation instructions and stop.

---

## Step 2: Project Style Detection & Confirmation

### Existing project

Scan the project for existing style configuration:

- Read `pyproject.toml` for `[tool.ruff]`, `[tool.mypy]`, `[tool.pyright]`
- Read `.editorconfig` if present
- Read `ruff.toml` if present
- Read test file headers in `tests/` to infer conventions

Then ask:

```
AskUserQuestion(
  "Existing configuration detected. How would you like to proceed?\n"
  "A) Keep existing style (recommended)\n"
  "B) Override with AutoFlow defaults\n"
  "C) Customize manually"
)
```

### New project

Ask the user to select a tech stack:

```
AskUserQuestion(
  "Select tech stack for this project:\n"
  "A) Recommended: Python 3.13 + uv + ruff + pyright + pre-commit + rich + make\n"
  "B) Conservative: Python 3.12 + pip + flake8 + mypy + pytest-html\n"
  "C) Custom: enter your preferences"
)
```

If the user selects C, ask follow-up questions for each component:
- Python version
- Package manager (uv / pip / poetry)
- Linter (ruff / flake8 / pylint)
- Type checker (pyright / mypy)
- Extras (pre-commit, rich, make)

Store the confirmed stack as `_stack` for use in later steps.

---

## Step 3: Source Code Repository Configuration

Ask for one or more repository URLs to clone as source-code reference:

```
AskUserQuestion(
  "Enter source repository URLs (one per line, include optional @branch suffix):\n"
  "Example:\n"
  "  https://git.example.com/group1/backend.git\n"
  "  https://git.example.com/group2/api.git@develop\n"
  "Leave blank when done."
)
```

For each URL, derive the local path automatically:

```
https://git.example.com/group1/repo1.git  →  .repos/group1/repo1/
https://git.example.com/group2/repo2.git@develop  →  .repos/group2/repo2/ (branch: develop)
```

Clone and checkout each repo:

```bash
git clone <url> <local_path>
git -C <local_path> checkout <branch>   # if branch specified
```

Then ask for URL-prefix-to-repo mapping so the HAR analyzer can resolve
which source repo owns each request:

```
AskUserQuestion(
  "Map URL prefixes to repos (one mapping per line):\n"
  "Format: <url-prefix> → <repo-name>\n"
  "Example:\n"
  "  /api/v1 → backend\n"
  "  /admin  → admin-portal\n"
  "Leave blank when done."
)
```

Generate `repo-profiles.yaml` at the project root:

```yaml
# repo-profiles.yaml — auto-generated by /using-autoflow
repos:
  - name: <repo-name>
    local_path: .repos/<group>/<repo>/
    remote: <url>
    branch: <branch>
    url_prefixes:
      - <prefix1>
      - <prefix2>
```

## Step 4: Connection Configuration

Ask for the base URL of the system under test:

```
AskUserQuestion(
  "Enter the base URL of the target system (e.g. http://172.16.115.247):"
)
```

Ask for the authentication method:

```
AskUserQuestion(
  "Select authentication method:\n"
  "A) Cookie (paste raw Cookie header value)\n"
  "B) Token (Bearer token)\n"
  "C) Username + Password"
)
```

Collect credentials based on the chosen method (one AskUserQuestion per field).

Ask about optional integrations:

```
AskUserQuestion(
  "Configure database connection? (y/n)\n"
  "If yes, you will be asked for host, port, user, password, and database name."
)
```

If yes, collect DB fields one by one.

```
AskUserQuestion(
  "Configure notification webhook? (y/n)\n"
  "Supported: DingTalk, Feishu, Slack"
)
```

If yes, ask for webhook URL and platform type.

Write `.env` and `.env.example`:

`.env` — contains real values (gitignored):
```
BASE_URL=<value>
AUTH_COOKIE=<value>        # or AUTH_TOKEN / AUTH_USER + AUTH_PASS
DB_HOST=<value>            # if configured
NOTIFY_WEBHOOK=<value>     # if configured
NOTIFY_PLATFORM=<value>    # dingtalk | feishu | slack
```

`.env.example` — placeholder values only (committed to VCS):
```
BASE_URL=http://your-server
AUTH_COOKIE=your_cookie_here
```

## Step 5: Scaffold Generation + CLAUDE.md

Run the scaffold script to lay out the project structure:

```bash
python3 ${CLAUDE_SKILL_DIR}/../../scripts/scaffold.py \
  --stack "${_stack}" \
  --base-url "${BASE_URL}"
```

The script creates:
```
tests/
  conftest.py
  test_smoke.py
Makefile          (if make in stack)
pyproject.toml    (updated)
.pre-commit-config.yaml  (if pre-commit in stack)
.gitignore        (ensures .env, .repos/ are ignored)
```

Auto-generate `CLAUDE.md` at the project root with the following sections:

```markdown
# CLAUDE.md — AutoFlow Project

## Tech Stack
<confirmed stack from Step 2>

## Project Structure
<directory tree>

## Source Code References
<list of repos from repo-profiles.yaml with local paths>

## Convention Index
- Tests: tests/
- Fixtures: tests/conftest.py
- Assertions: see prompts/assertion-layers.md
- Code style: see prompts/code-style-python.md

## Environment
- Base URL: ${BASE_URL}
- Auth: <method>
- DB: <configured / not configured>
- Notifications: <configured / not configured>
```

Finally, print an initialization summary:

```
AutoFlow initialized successfully
──────────────────────────────────────────
Stack:        Python 3.13 + uv + ruff + pyright
Repos cloned: 2
URL mappings: 3
Auth method:  Cookie
DB:           configured
Webhook:      Feishu
──────────────────────────────────────────
Next step: run /autoflow <path-to.har>
```
